Guia rápido de uso do painel administrativo.

Passo a passo: onboarding de um novo cliente
  1. Base Operacional — Crie a base do cliente pelo formulário "Criar base operacional". É aqui que os dados da agenda ficam.
  2. Clientes — Cadastre os dados comerciais (razão social, CNPJ, e-mail, logomarca) e vincule à base criada no passo anterior.
  3. Vigências — Defina o plano e o período de uso da licença.
  4. Compradores — Cadastre os usuários do cliente diretamente no Supabase (tabela compradores) ou via importação, informando nome, e-mail e tenant_id da base.
  5. Enviar Convites — Na seção Base Operacional, clique em "Enviar Convites". O comprador recebe um e-mail com o link para definir a senha (24h) e o instalador de atalho para Windows (arquivo .bat).
  6. Abrir Portal — Use "Abrir Portal" para verificar o ambiente do cliente antes de entregar.

💡 O comprador instala o atalho clicando duas vezes no arquivo instalar_atalho.bat que recebe no e-mail. O atalho abre o sistema em modo app (sem barra do navegador).

O que é a Base Operacional e o que é Cliente?

São dois registros separados com finalidades distintas:

  • Base Operacional (tenant) — onde os dados da agenda ficam: fornecedores, compradores, ocorrências. É o ambiente ativo do cliente.
  • Cliente — os dados comerciais: razão social, CNPJ, contato, vigência. É quem você fatura.

Um cliente deve sempre estar vinculado a uma base operacional para funcionar corretamente.

💡 Crie sempre a Base Operacional antes de cadastrar o Cliente.

Como enviar convite de acesso para um comprador?
  1. Vá em Base Operacional e localize o tenant do cliente.
  2. Clique em Enviar Convites.
  3. O sistema lista os compradores cadastrados. Digite o e-mail do comprador desejado e confirme.
  4. O comprador recebe um e-mail com dois itens: o link para definir a senha (válido 24h) e o botão "Baixar instalador do atalho" para Windows.

O que o comprador faz ao receber o e-mail

  1. Clica em "Criar minha senha" → define a senha na página de configuração de acesso → entra no portal automaticamente já selecionado como comprador ativo
  2. Clica em "Baixar instalador do atalho" → abre o arquivo instalar_atalho.bat com duplo clique → atalho criado na área de trabalho com o ícone do sistema

⚠️ O comprador precisa ter e-mail cadastrado na tabela compradores para receber o convite.

Como gerenciar administradores do painel?

Apenas o administrador master pode convidar, revogar ou excluir outros admins.

  1. Acesse a seção Admins.
  2. Para convidar: preencha o e-mail e nome → "Enviar convite". O novo admin recebe e-mail com link para definir senha e instalar o app.
  3. Para revogar: clique em "Revogar acesso" — o usuário perde o acesso ao painel mas a conta permanece.
  4. Para excluir: clique em "Excluir" — remove o usuário permanentemente do Supabase.

⚠️ O administrador master não pode ser revogado ou excluído.

O que faz o botão "Abrir Portal"?

Abre o portal do cliente em uma nova aba já autenticado como administrador. Útil para:

  • Verificar se a agenda está carregando corretamente
  • Ajudar o cliente com dúvidas de uso
  • Validar o onboarding antes de entregar o acesso

Comportamento atual

  • O feedback mostra o nome do cliente sendo aberto: "Gerando acesso ao portal de “Grupo X”..."
  • A nova aba recebe foco automaticamente após abrir
  • A autenticação é reutilizada por até 55 minutos (sem chamada extra ao Supabase)
  • Cada aba abre um cliente isolado — abrir Velanes não interfere na aba do SV já aberta

⚠️ A sessão "Abrir Portal" expira em 1 hora. Para continuar, feche a aba e clique em "Abrir Portal" novamente.

📲 Instalador de Atalho Windows — como funciona?

O e-mail de convite inclui um botão para baixar o instalador de atalho (instalar_atalho.bat). Quando o comprador clica duas vezes nesse arquivo:

  1. O instalador detecta automaticamente se Edge ou Chrome está instalado (Edge tem prioridade)
  2. Baixa o ícone do sistema do servidor
  3. Cria um atalho na área de trabalho que abre o portal em modo app — sem barra do navegador, igual a um programa instalado

URL de redirecionamento

O atalho aponta para agenda-de-compras-api.vercel.app/portal, que redireciona para o portal real. Se o endereço do portal mudar, basta atualizar a variável FRONTEND_URL no Vercel — todos os atalhos já instalados continuam funcionando sem redistribuição.

⚠️ Requer Microsoft Edge ou Google Chrome. Não funciona com Firefox ou Internet Explorer.

Senha da auditoria — como funciona?

A senha da auditoria protege o modo de auditoria no portal do cliente. Quando o comprador clica em "Auditoria", precisa digitar essa senha.

Defina ou altere pelo botão Senha da auditoria em cada cliente na seção Clientes.

💡 A senha padrão é service. Recomendamos trocar após o onboarding.

📧 Relatório diário por e-mail — como ativar?

O sistema envia automaticamente um relatório por e-mail toda noite útil (seg–sex) às 21h BRT. Para ativar:

  1. Base Operacional — Localize o tenant e marque o toggle "Envio de relatório diário". Enquanto desativado, nenhum e-mail é enviado para aquele cliente.
  2. Abrir Portal — Acesse o portal do cliente e vá em Compradores.
  3. Edite cada comprador que deve receber o e-mail e marque as opções de notificação desejadas (veja tópico abaixo).

Conteúdo do relatório

  • Itens em atraso — agenda de compras pendente com data vencida (destaque vermelho)
  • Agenda do próximo dia útil — compras agendadas + outros compromissos
  • Tratamentos do dia anterior — o que foi realizado, com justificativas quando registradas
  • KPIs do mês corrente e anterior — 10 métricas: realizadas, atrasadas, postergadas, antecipadas, adiadas, parâmetros alterados e execuções fora da carteira

O e-mail inclui um PDF anexo com o mesmo conteúdo formatado para impressão.

💡 O relatório só é enviado para compradores com e-mail cadastrado e com pelo menos uma das opções de notificação ativa.

📋 Notificações por comprador — gestor, auditoria e agenda

Cada comprador tem três configurações independentes de notificação, editáveis em Compradores no portal do cliente:

CampoO que recebePara quem
Gestor Dados consolidados de todos os compradores do tenant Gerente / responsável geral
Receber auditoria Tratamentos realizados no dia anterior (com justificativas) Qualquer comprador com acesso ao e-mail
Receber agenda próximo Itens em atraso + agenda do próximo dia útil Qualquer comprador com acesso ao e-mail

Diferença entre Gestor e comprador comum

  • Gestor: recebe KPIs e detalhes de toda a equipe — ideal para supervisores
  • Comprador comum: recebe apenas os dados da própria carteira de fornecedores
  • É possível ter mais de um gestor por tenant

Como configurar

  1. No painel admin, clique em Abrir Portal do cliente desejado
  2. No portal, acesse o menu Compradores
  3. Clique em editar no comprador e marque as checkboxes Gestor, Receber auditoria e/ou Receber agenda próximo
  4. Salve — a configuração entra em vigor no próximo envio noturno

⚠️ Apenas admin_client e admin_portal podem alterar essas configurações. Compradores comuns não têm acesso a esses campos.
