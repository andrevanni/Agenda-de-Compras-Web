Ajuda e referências

Guia rápido de uso do painel administrativo.

Como cadastrar um novo cliente?

O fluxo correto de onboarding segue esta ordem:

  1. Base Operacional — A base já é criada automaticamente quando o cliente começa a usar o sistema. Ela aparece na seção "Base Operacional" após o primeiro acesso.
  2. Clientes — Cadastre os dados comerciais do cliente (razão social, CNPJ, e-mail) e vincule à base operacional correspondente.
  3. Vigências — Defina o período de uso e o status da licença.
  4. Convites — Na seção "Base Operacional", clique em "Enviar Convites" para enviar o link de acesso por e-mail ao comprador.
O que é a Base Operacional?

A Base Operacional (tenant) é o isolamento de dados de cada cliente no banco. Todos os dados — compradores, fornecedores, agenda — ficam separados por base operacional.

O Cliente (na aba Clientes) são os dados comerciais: CNPJ, razão social, contato. Um cliente deve estar sempre vinculado a uma base operacional.

💡 Se um cliente não aparece vinculado, edite o cadastro dele e selecione a base no campo "Base operacional vinculada".

Como enviar convite de acesso para um comprador?
  1. Vá em Base Operacional e localize o tenant do cliente.
  2. Clique em Enviar Convites.
  3. O sistema lista os compradores cadastrados na base. Digite o e-mail do comprador que deseja convidar.
  4. O comprador receberá um e-mail com um link para definir a própria senha.

⚠️ O comprador precisa ter um e-mail cadastrado no sistema para receber o convite.

O que faz o botão "Abrir Portal"?

O botão Abrir Portal abre o portal do cliente em uma nova aba, já autenticado como administrador. Você pode navegar pelo sistema como se fosse o próprio cliente.

Isso é útil para:

  • Verificar se a agenda está carregando corretamente
  • Ajudar o cliente com dúvidas de uso
  • Validar o onboarding antes de entregar o acesso

⚠️ Requer que o backend esteja configurado e o Token Administrativo esteja correto em "Conexão avançada".

Status de vigência — o que significa cada um?
StatusSignificado
AtivoCliente com acesso liberado normalmente.
ImplantaçãoOnboarding em andamento. Acesso liberado mas ainda em configuração.
VencidoLicença expirada. Acesso deve ser renovado.
BloqueadoAcesso suspenso manualmente.
Senha da auditoria — como funciona?

A senha da auditoria protege o acesso ao modo de auditoria dentro do portal do cliente. Quando o comprador clica em "Auditoria", precisa digitar essa senha.

Você pode definir ou alterar a senha pelo botão Senha da auditoria em cada cliente na seção Clientes.

💡 A senha padrão é service. Recomendamos trocar após o onboarding.

Conexão avançada — quando preciso mexer?

A Conexão avançada guarda as credenciais de acesso ao banco e ao backend. Você só precisa alterar se:

  • Trocar de projeto Supabase
  • Trocar a URL do backend (Railway, Vercel, etc.)
  • Resetar o Token Administrativo

⚠️ O Token Administrativo deve ser o mesmo valor de ADMIN_API_TOKEN configurado no backend.
