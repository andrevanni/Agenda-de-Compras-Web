Guia rápido de uso do painel administrativo.

Passo a passo: onboarding de um novo cliente
  1. Base Operacional — Crie a base do cliente pelo formulário "Criar base operacional". É aqui que os dados da agenda ficam.
  2. Clientes — Cadastre os dados comerciais (razão social, CNPJ, e-mail, logomarca) e vincule à base criada no passo anterior.
  3. Vigências — Defina o plano e o período de uso da licença.
  4. Compradores — Cadastre os usuários do cliente diretamente no Supabase (tabela compradores) ou via importação, informando nome, e-mail e tenant_id da base.
  5. Enviar Convites — Na seção Base Operacional, clique em "Enviar Convites" para o cliente receber o link de acesso por e-mail.
  6. Abrir Portal — Use "Abrir Portal" para verificar o ambiente antes de entregar ao cliente.
O que é a Base Operacional e o que é Cliente?

São dois registros separados com finalidades distintas:

  • Base Operacional (tenant) — onde os dados da agenda ficam: fornecedores, compradores, ocorrências. É o ambiente ativo do cliente.
  • Cliente — os dados comerciais: razão social, CNPJ, contato, vigência. É quem você fatura.

Um cliente deve sempre estar vinculado a uma base operacional para funcionar corretamente.

💡 Crie sempre a Base Operacional antes de cadastrar o Cliente.

Como enviar convite de acesso para um comprador?
  1. Vá em Base Operacional e localize o tenant do cliente.
  2. Clique em Enviar Convites.
  3. O sistema lista os compradores cadastrados. Digite o e-mail do comprador desejado.
  4. O comprador recebe um e-mail com link para definir a senha e instalar o app.

⚠️ O comprador precisa ter e-mail cadastrado na tabela compradores para receber o convite.

Como gerenciar administradores do painel?

Apenas o administrador master pode convidar, revogar ou excluir outros admins.

  1. Acesse a seção Admins.
  2. Para convidar: preencha o e-mail e nome → "Enviar convite". O novo admin recebe e-mail com link para definir senha e instalar o app.
  3. Para revogar: clique em "Revogar acesso" — o usuário perde o acesso ao painel mas a conta permanece.
  4. Para excluir: clique em "Excluir" — remove o usuário permanentemente do Supabase.

⚠️ O administrador master não pode ser revogado ou excluído.

O que faz o botão "Abrir Portal"?

Abre o portal do cliente em uma nova aba já autenticado como administrador. Útil para:

  • Verificar se a agenda está carregando corretamente
  • Ajudar o cliente com dúvidas de uso
  • Validar o onboarding antes de entregar o acesso
Status de vigência — o que significa cada um?
StatusSignificado
AtivoCliente com acesso liberado normalmente.
ImplantaçãoOnboarding em andamento. Acesso liberado mas ainda em configuração.
VencidoLicença expirada. Acesso deve ser renovado.
BloqueadoAcesso suspenso manualmente.
Senha da auditoria — como funciona?

A senha da auditoria protege o modo de auditoria no portal do cliente. Quando o comprador clica em "Auditoria", precisa digitar essa senha.

Defina ou altere pelo botão Senha da auditoria em cada cliente na seção Clientes.

💡 A senha padrão é service. Recomendamos trocar após o onboarding.

Conexão avançada — quando preciso mexer?

Guarda as credenciais do banco e do backend. Altere apenas se:

  • Trocar de projeto Supabase
  • Trocar a URL do backend

Instale o app Admin

Acesse o painel administrativo direto da área de trabalho, sem precisar do navegador.

Como instalar

Chrome / Edge: ícone ⊕ na barra → "Instalar"

Menu: três pontos ⋮ → "Instalar Agenda de Compras – Admin"

iPhone/iPad: compartilhar ⎙ → "Adicionar à Tela de Início"
